🖼️ Topic 1: Graphics

① What is a Bitmap Image?

A bitmap image (also called a raster image) is made up of a grid of tiny coloured squares called pixels (picture elements). Each pixel stores exactly one colour value. When you zoom into a photograph, you can see these individual pixels — the image looks blocky. This is the fundamental nature of bitmaps: they store every colour, dot by dot, in a fixed grid.

Common bitmap file formats include .jpg, .png, and .bmp. The .png format is special because it supports transparent backgrounds using an alpha channel — this is why logos and icons on websites often use PNG. Note: .psd (Photoshop) files also support transparency.

Warning: Pixelation

When you enlarge a bitmap graphic, the computer stretches the existing pixels to fill the bigger space. Each pixel becomes a larger coloured square, creating the blurry, blocky effect called pixelation. This cannot be fixed — you cannot recover detail that was never stored. This is why you should never enlarge a bitmap beyond its original size.

② Resolution and PPI

Resolution refers to the number of pixels stored in an image, often measured in ppi (pixels per inch). A higher resolution means more pixels per inch, which means a sharper image — but also a larger file size.

  • 72 ppi — Standard screen/web resolution. Extra pixels beyond this are invisible on screen and just waste storage.
  • 300 ppi — Professional print standard. Below this, individual pixels become visible to the naked eye when printed.

Key rule: Increasing the resolution of an image increases the number of pixels stored, which directly increases the file size.

Common Confusion: Resolution vs. Quality

Resolution (ppi) is NOT the same as image quality. A photo can have high ppi but still look bad if it was originally taken in poor lighting or on a low-quality camera. Resolution only measures how densely packed the pixels are — not how good the original image was.

③ Calculating File Size

This is one of the most frequently tested calculation topics. You MUST know the formula:

File size (bits) = Width (px) × Height (px) × Colour depth (bits per pixel) → ÷ 8 to convert bits to Bytes → ÷ 1,024 to convert Bytes to Kilobytes (KB) → ÷ 1,024 again to convert Kilobytes to Megabytes (MB)
Worked Example 1

An image is 400 × 300 pixels with 24-bit colour depth. What is the file size in KB?

Step 1: Total pixels = 400 × 300 = 120,000 pixels
Step 2: Total bits = 120,000 × 24 = 2,880,000 bits
Step 3: Bytes = 2,880,000 ÷ 8 = 360,000 bytes
Step 4: KB = 360,000 ÷ 1,024 = ≈ 351.6 KB

Worked Example 2 — Full HD Image

A 1920 × 1080 pixel image uses 24-bit colour. File size in MB?

1920 × 1080 = 2,073,600 px
× 24 = 49,766,400 bits
÷ 8 = 6,220,800 bytes
÷ 1,024 = 6,075 KB
÷ 1,024 = ≈ 5.93 MB

Most Common Mistake in File Size Questions

The formula uses bits per pixel, not bytes. Always multiply W × H × colour depth FIRST to get bits, THEN divide by 8 to get bytes. Students who divide by 8 too early or forget to convert get the wrong answer.

④ Colour Depth

Colour depth is the number of bits used to represent the colour of a single pixel. The formula for the number of available colours is:

⑤ What is a Vector Image?

Unlike bitmaps, vector graphics do not store a grid of pixels. Instead, they store shapes as mathematical geometric descriptions — circles, lines, polygons, and curves defined by coordinates, sizes, and properties.

For each shape object, the computer stores properties such as:

  • The type of shape (circle, rectangle, line, polygon)
  • Its position — centre point coordinates (x, y)
  • Its dimensions — radius, width, height
  • Its fill colour and outline (line) colour

When the image is scaled, the computer simply recalculates all the mathematical descriptions at the new size. This is why vector images are perfectly sharp at any size — they never pixelate. They are also usually much smaller in file size than bitmaps because the computer only stores mathematical properties rather than millions of pixel values.

Common uses of vector graphics: logos, clip art, animations, text characters, icons. Text characters are vector-based, which is why they look sharp at any font size.

When to Use Vector vs Bitmap
  • Vector: logos, icons, clip art, animations, text, flags, simple diagrams — shapes with flat colours that need to scale
  • Bitmap: photographs, realistic images, screenshots — complex images with millions of subtle colour variations

Saving a JPEG file multiple times degrades quality further each time, because each save re-applies lossy compression to the already-degraded image. Always work from an original lossless format (.png or .bmp) and export as JPEG only for the final version.

⑧ Conveying Meaning Through Design

Design choices communicate meaning to an audience beyond just visual appearance. Every decision sends a message:

  • Font style: Serif fonts (with small "feet" at character ends — like Times New Roman) convey tradition, formality, and authority. Sans Serif fonts (plain — like Arial) convey modernity, simplicity, and cleanliness. Keep the number of different fonts to a minimum — using too many creates visual chaos and looks unprofessional.
  • Font size: Larger text signals more importance. Used to create visual hierarchy — guiding the viewer's eye to what matters most first.
  • Colour schemes: Red = urgency/danger. Blue = trust/calm/professionalism. Green = nature/health. Yellow = happiness/warning. These are deliberate psychological associations designers use to set tone and mood.
  • Layers: When creating complex graphics in software like Photoshop, using layers lets you manipulate different parts of an image independently — editing one element without accidentally affecting others. Essential for professional design work.
Extended Knowledge: Rule of Thirds

The Rule of Thirds divides a frame into a 3x3 grid using two horizontal and two vertical lines. Placing the main subject at one of the four intersection points (rather than dead centre) creates more dynamic and visually interesting compositions. This principle applies to photography, film, video, and digital design. It is a foundational concept in visual communication.

Hub vs Switch — Easy to Confuse

A hub sends data to ALL computers connected to it — even if only one computer needs it. A switch sends data only to the ONE correct computer. The switch is far more efficient and secure. Modern networks always use switches, not hubs.

③ Network Topologies

A topology is the physical layout of how devices are connected in a network.

TopologyStructureAdvantageDisadvantage
BusAll computers connect to one central cable with "terminators" at each end to stop signals bouncing backCheap — requires very little cableIf the main cable breaks, the ENTIRE network fails
StarEvery computer connects individually to a central switch — the most common modern layoutIf one cable breaks, only that computer is affected; all others still workIf the central switch fails, the whole network goes down
RingComputers connected in a closed loop; data travels one-way around the ring using a "token"Equal access for all devices; no data collisionsOne break disrupts the whole ring
Why Star is the Most Common Topology Today

Star topology dominates modern networks because: one cable failure only affects one device (not the whole network), it is easy to add new computers (just plug one cable into the switch), and faults are easy to identify and fix. Schools and offices almost universally use star topology.

④ Bandwidth, Speed, and the 8-Bit Rule

Bandwidth refers to the volume of data that can be carried at one time — often compared to the width of a water pipe. More bandwidth = more data flows simultaneously. Connection speeds are measured in Mbps (Megabits per second).

CRITICAL: The 8-Bit Rule for Download Time

File sizes are measured in Bytes (capital B). Connection speeds are measured in bits (lowercase b). These are DIFFERENT units. You MUST convert before calculating.

Download time (seconds) = (File size in MB × 8) ÷ Speed in Mbps

Why multiply by 8? Because 8 bits = 1 Byte. So 1 MB = 8 Mb (megabits).

Worked Example: Download Time

How long to download a 500 MB file on a 100 Mbps connection?

Step 1: Convert file size from MB to Mb → 500 × 8 = 4,000 Mb
Step 2: Divide by connection speed → 4,000 ÷ 100 = 40 seconds

Buffering occurs when a video plays faster than it downloads. The computer must pause to pre-load ("buffer") a waiting room of data before it can resume playing. This happens when your connection speed (bandwidth) is too slow for the video's data rate.

⑥ The Internet, WWW, and URLs

These two terms are often confused but they mean very different things:

  • The Internet — The physical hardware: millions of interconnected networks of computers, cables, routers, servers, and other devices worldwide. It is the infrastructure.
  • The World Wide Web (WWW) — The collection of websites and web pages stored on computers connected to the internet. The web runs ON TOP of the internet — it is one of many services the internet provides (alongside email, file transfer, etc.).
Anatomy of a URL

http://www.wildlife.org.au/images/koala.jpg

  • http — Protocol: how the data is transferred (HyperText Transfer Protocol)
  • www — Subdomain: indicates the World Wide Web
  • wildlife — Domain name: the website's unique name
  • .org — Top-level domain: indicates the type of organisation (non-profit)
  • .au — Country code: Australia
  • /images/koala.jpg — File path: the specific file location on the server

DNS (Domain Name Server): Computers communicate using numeric IP addresses (like 173.194.34.95), but humans find these impossible to remember. The DNS acts like a giant phone book — it translates easy-to-read domain names (like google.com) into the IP addresses that computers actually use to find each other.

⑦ Packet Switching — How Data Travels

Data across the internet is not sent as one continuous stream. Instead, it is broken into small chunks called packets. Each packet:

  • Contains a piece of the data PLUS addressing information (where it came from, where it is going, its order number)
  • Takes the fastest available route at that moment — not necessarily the shortest path
  • May travel a different route from other packets belonging to the same message
  • Is reassembled in the correct order at the destination using the order numbers

⑨ Network Security and Encryption

Encryption is the process of converting readable data (Plaintext) into unreadable scrambled code (Ciphertext) to prevent unauthorised access during transmission. Without encryption, anyone intercepting your data packets could read them directly.

Encryption requires two components working together:

  • Algorithm — The set of rules / mathematical process used to scramble the data
  • Key — The secret password/value used to lock (encrypt) and unlock (decrypt) the data

HTTPS: The "S" in HTTPS stands for Secure. Websites using HTTPS display a padlock icon in the browser address bar, proving that all data exchanged between your browser and the server is encrypted using TLS (Transport Layer Security).

HTTPS Misconception

The padlock icon means your CONNECTION is encrypted. It does NOT mean the website itself is safe, honest, or trustworthy. A scam website can also use HTTPS. Always check the actual domain name carefully — do not just look for the padlock.

🤖 Topic 3: Artificial Intelligence (AI)

① What is Artificial Intelligence?

In simple terms, AI is making computers smart enough to do things that normally require human intelligence. This includes learning from information, solving complex puzzles, and recognising patterns that are difficult or impossible to program manually with fixed rules.

② Traditional AI vs Generative AI

FeatureTraditional AI — "The Doer"Generative AI — "The Creator"
Design purposeDesigned for ONE specific task; responds based on fixed data or pre-set rulesLearns from massive datasets to generate brand new content that did not previously exist
What it doesOrganises or responds to existing information — it does NOT create anything newCreates new original text, images, music, code, and other content
ExamplesEmail spam filters, GPS route finders, Netflix recommendations, chess engines, facial recognition, fingerprint scanners, game AI opponentsChatGPT / Claude (text), Midjourney / DALL-E (images), AI music composers
Common Mistake: Thinking all AI is the same

Traditional AI and Generative AI are fundamentally different. A spam filter (Traditional AI) does NOT create content — it classifies existing emails as spam or not-spam. ChatGPT (Generative AI) generates new text it has never seen before. Understanding this distinction is essential for exam questions.

③ Machine Learning (ML) — How AI Learns

Machine learning is the process of teaching an AI by showing it thousands of examples, rather than writing a specific rule for every possible scenario. The computer analyses these examples and looks for mathematical patterns.

Key vocabulary you must know:

  • Dataset — The library of examples used to teach the AI. Like the textbooks the AI studies from. A good dataset is large AND diverse.
  • Features — Specific identifiable clues that AI engineers tell the AI to look for. For example, to distinguish cats from dogs: "pointy ears" vs "floppy ears"; "vertical pupils" vs "round pupils."
  • Training — The process of the AI studying the dataset and repeatedly adjusting its internal settings to improve its accuracy at the task.
  • Model — The final result once training is complete. The model is the "brain" that is now an expert at the specific task. Think of it as the AI having finished all its studying and graduated.
Why Diverse Datasets Matter — AI Bias

For an AI to work fairly for everyone, it must be trained on diverse data that represents the full range of real-world cases. An AI trained only on one demographic will perform poorly on others. For example: a facial recognition AI trained mostly on light-skinned faces performs poorly on darker skin tones. An AI hiring tool trained on 10 years of historical hiring data — which reflected past gender bias — learned to downgrade women's CVs. Biased data in = biased decisions out.

④ Large Language Models (LLMs) and Tokenization

An LLM (Large Language Model) is a type of machine learning AI that specialises in language. They are "Large" because they are trained on billions of words from the internet, books, and other text sources.

Tokenization: Computers cannot process raw text directly. They break text down into smaller, meaningful pieces called tokens — which can be words, prefixes, suffixes, or word fragments.

Why Tokenization is Efficient

By breaking "unhappy" into tokens "un-" + "happy," the AI can understand new compound words it has never seen before by combining known parts. This is far more efficient than storing every possible word as a separate unit — especially across multiple languages.

LLMs are Prediction Engines: An LLM does NOT "understand" the world the way humans do. It mathematically calculates which token is most likely to come next, based on all the statistical patterns in its massive training data. It is a super-powered next-token predictor — not a thinking machine.

Extended Knowledge: AI Hallucination

Because LLMs predict likely text rather than look up verified facts, they can "hallucinate" — generate confident-sounding but completely false information. They can invent citations, statistics, dates, and events that never happened, and present them as fact. This is why you should ALWAYS verify important information generated by AI from reliable, authoritative sources.

⑤ Real-World AI Applications

Key historical milestones in AI development:

  • ELIZA (1960s) — An early chatbot that simulated conversation using pattern matching and template responses. It convinced some people it was human — an early example of AI passing for human interaction.
  • IBM Deep Blue (1997) — Defeated world chess champion Garry Kasparov. A landmark moment — traditional rule-based AI proving it could outperform the best human at a complex intellectual game.
  • Robot vacuum cleaners — AI-driven autonomous machines for everyday home use, using sensors and decision rules to navigate spaces.

You almost certainly interact with AI every single day:

  • Facial / fingerprint recognition on your smartphone — Traditional AI classifying your biometric data
  • Speech recognition in digital assistants (Siri, Alexa, Google Assistant) — ML-based language understanding
  • Computer opponents in video games — Rule-based Traditional AI using decision trees and state machines
  • GPS route finders — Optimising routes in real time based on traffic data
  • Netflix / Spotify recommendations — Pattern-matching your viewing/listening history to suggest similar content

⑥ Decision Rules in AI Systems

Some AI, like that used in self-driving cars, combines complex sensors with decision rules:

Self-Driving Car Decision Rules

"If there is an obstacle ahead AND no room to either side, THEN stop."
"If the traffic light is red, THEN stop."
"If pedestrian detected in path, THEN slow down and check."

These conditional rules combine with data from cameras, lidar (laser distance sensors), and radar to make real-time driving decisions. This is how complex AI systems handle real-world unpredictability.

⑦ AI Ethics and Academic Honesty

Academic honesty means presenting your own work and ideas, and giving proper credit when you use the work of others. Using AI irresponsibly violates this principle.

Acceptable AI Use in SchoolUnacceptable AI Use in School
Using AI to brainstorm ideas before writing yourselfCopying a full AI-generated essay and submitting it as your own
Using AI to explain a concept you do not understandUsing AI to bypass critical thinking or skip the learning process
Using AI to check your grammar after writingSubmitting AI-generated code without understanding what it does
Why submitting AI work as your own is wrong

When you submit AI-generated work as your own, you are not doing the learning — the AI is. The entire purpose of assignments is to develop your own understanding and skills. Copying AI output means you miss the learning AND misrepresent your abilities to your teacher. This is academic dishonesty regardless of whether it is detected.

🐍 Topic 4: Python Programming

① Introduction to Python and IDLE

Python is one of the world's most popular programming languages, used by organisations like NASA and Google, in professional software such as Instagram and Spotify, and in major video games. It is designed to be readable and accessible, making it an excellent language to learn first.

Python uses a development environment called IDLE, which has two modes:

  • Interactive Mode: Type one line of code and see the result immediately. Good for testing small things and experimenting.
  • Script Mode: Write and save a full program in a .py file so it can be run again later. Used for all real programs.

Two fundamental built-in functions you will use in almost every program:

  • print() — Displays information on the screen for the user to read
  • input() — Displays a prompt, waits for the user to type something, and captures their response. ALWAYS returns a string — no matter what the user types.

② Variables and Naming Rules

A variable is a named location in the computer's memory used to temporarily store a value (text or a number). Think of it as a labelled box — you can put something in it, change what is in it, and look at what is inside.

Creating variables

name = "Aisha" # stores text
age = 14 # stores a whole number
score = 87.5 # stores a decimal number
passed = True # stores True or False

Variable Naming Rules in Python:

  • Can only contain letters, numbers, and underscores: my_score, player1
  • CANNOT start with a number: 1name is invalid and causes an error
  • CANNOT use Python reserved words: class, if, for, while, print, input — these have special meanings in Python and cannot be used as variable names
  • Case matters: Score and score are treated as two completely different variables

Casting is the process of manually converting one data type to another using built-in functions. This is frequently necessary because input() ALWAYS returns a string.

Type Casting Examples

CRITICAL: input() always returns a STRING

Even if the user types 42, Python stores it as the text '42', not the number 42. You CANNOT perform maths on it directly — you will get a TypeError. You MUST cast it first using int() or float(). This is one of the most common beginner mistakes.

BIDMAS — Python follows the same order of operations as mathematics: Brackets, Indices (powers/exponentiation), Division/Multiplication, Addition/Subtraction. Always use brackets to make complex expressions clear and unambiguous.

^ is NOT power in Python!

In maths notation, ^ often means "to the power of." In Python, ^ is the bitwise XOR operator — it does something completely different and will give you a wrong answer. For powers, always use **: 2**8 = 256. This confuses many students who have used ^ in other contexts.

⑤ Selection — if / elif / else

Selection allows a program to make decisions and take different paths based on conditions. Without selection, programs would do exactly the same thing every single time — not very useful!

Full if / elif / else structure

score = int(input("Enter your score: "))

if score >= 70: # condition — must end with colon
    print("Pass") # must be indented 4 spaces
elif score >= 50: # "else if" — another condition to check
    print("Merit")
else: # catch-all — runs if NOTHING above was True
    print("Fail")

Python checks conditions top-to-bottom and stops at the FIRST one that is True. The else block has no condition — it is the default that runs when nothing else matched.

Comparison operators used in conditions:

The Three Most Common Python Bugs
  • Missing colon — Every if, elif, else, for, while, and def line MUST end with a colon :. Forgetting it causes a SyntaxError immediately.
  • Wrong indentation — Python uses whitespace (4 spaces recommended) to define code blocks. Inconsistent or missing indentation causes an IndentationError and the program cannot run. Unlike most languages, Python has NO curly braces — indentation IS the structure.
  • Using = instead of == — = assigns a value to a variable. == compares two values. Using = inside an if condition is a SyntaxError. This is one of the most frequent mistakes for beginners.

🔒 Topic 5: Cyber Security

① What is Cybercrime?

Cybercrime refers to any crime committed using a computer or a network. It is one of the fastest-growing criminal industries globally — making more money for criminals than illegal drug trafficking. Someone's identity is stolen as often as every three seconds.

The Computer Misuse Act (1990) is the primary law governing computer use in the UK. Under this act, it is illegal to:

  • Access (look at) someone else's computer files without their permission — even just looking without changing anything
  • Modify (change, delete, or add to) someone's files without their permission
  • Access files with the intent to commit a further crime (e.g., using data for fraud)

How to Spot a Scam Email — Red Flags Checklist
  • Non-personalised greeting — "Dear Customer," "Dear User," or "Hello Friend" instead of your actual name. Real companies know who you are.
  • Suspicious sender address — Variations that look almost right but are not: paypa1.com, amazon-security.net, support@app1e.com. Always check the full email domain carefully.
  • False sense of urgency — "Your account will be deleted in 24 hours," "Respond immediately," "Limited time offer." This pressure is designed to make you act without thinking carefully.
  • Poor spelling and grammar — Professional companies employ teams to proofread all communications. Multiple errors are a strong warning sign.
  • Forged links — The link text can be completely different from where it actually goes. Always hover your mouse cursor over a link to see the real destination URL before clicking. If you are unsure, go directly to the website by typing the address yourself.

③ Hacking and Types of Malware

Hacking is the illegal act of accessing or modifying computer files without permission. Hackers use several methods to gain unauthorised access:

  • Exploiting security holes (vulnerabilities) in software that has not been updated
  • Taking advantage of weak or obvious passwords
  • Using malware to gain a foothold in a system
  • Social engineering — tricking people into revealing passwords or clicking malicious links

Logic Bomb: A specific type of attack where a set of malicious instructions is hidden inside a legitimate system and only activates at a certain time or under specific conditions. Often used by disgruntled employees for revenge, or by blackmailers as a threat.

④ Encryption — Protecting Data

Encryption is the process of masking data by converting readable text (Plaintext) into unreadable scrambled code (Ciphertext) to prevent unauthorised access. Even if an attacker intercepts the encrypted data, they cannot read it without the correct key.

The Lock and Key Analogy

Think of encryption like putting a message in a padlocked box and sending it through the mail:

  • Algorithm = The type of padlock — the set of mathematical rules used to scramble the data
  • Key = The combination or key — the secret value used to lock (encrypt) and unlock (decrypt) the data
  • Without the correct key, the padlocked box (ciphertext) is unreadable — even if someone steals it in transit
The Shift (Caesar) Cipher — A Simple Encryption Example

One of history's oldest known ciphers — used by Julius Caesar for military communications approximately 2,000 years ago.

How it works: Shift each letter a fixed number of positions forward in the alphabet.

With shift key = 3: A→D, B→E, C→F, ..., X→A, Y→B, Z→C (wraps around)

Encrypt "HELLO" with shift 3:
H(8)+3=K, E(5)+3=H, L(12)+3=O, L+3=O, O(15)+3=R → ciphertext: "KHOOR"

Decrypt "KHOOR" with shift 3 — subtract 3 from each letter:
K(11)−3=H, H(8)−3=E, O(15)−3=L, O−3=L, R(18)−3=O → plaintext: "HELLO"

Wrap-around rule: When you reach Z and need to go further, wrap back to A. So Z+1=A, Z+2=B, Z+3=C.

HTTPS: The "S" stands for Secure. Websites using HTTPS display a padlock icon in the browser address bar. This indicates that all data exchanged between your browser and the server is encrypted. Always check for HTTPS before entering passwords, personal information, or payment details on any website.

⑤ Protection Strategies

Password Security Best Practices:

  • Avoid common, weak passwords that are tried first in every attack: "123456", "password", "password1", "qwerty", your name, or your birthdate
  • Use unique passwords for every different website — if attackers steal credentials from one site, they automatically try the same username and password on banking, email, and other accounts (this is called "credential stuffing")
  • Change passwords regularly, and immediately if you suspect they may have been compromised
  • A strong password uses 12+ characters, mixed uppercase and lowercase, numbers, AND symbols

Technical Defences:

  • SPAM filter — Automatically identifies and moves suspicious emails before they reach your inbox, reducing your exposure to phishing and malware
  • Keep browser and software up to date — Software updates include security patches that fix known vulnerabilities. Attackers actively scan for systems running old, unpatched software to exploit
  • Install reputable antivirus and antispyware software — Scans for, detects, and removes malicious software. Must be from a trusted company and kept regularly updated to recognise new threats

Safe Browsing Habits:

  • Never click links or open attachments in emails unless you are 100% certain they are genuine — even from addresses that look familiar
  • Avoid illegal sites that offer copyrighted material (films, software, music) for free — these sites are extremely high-risk sources of malware
  • Always verify that a website uses HTTPS before entering any personal information or payment details
  • Hover over links to check the real destination URL before clicking — the displayed text can be completely different from where it actually takes you
Extended Knowledge: Two-Factor Authentication (2FA)

Two-Factor Authentication adds a second layer of security beyond just a password. Even if an attacker steals your password (through a data breach or phishing), they still cannot access your account without the second factor. The three types of authentication factors are: something you know (password), something you have (a code sent to your phone, an authenticator app), or something you are (fingerprint, face recognition). Always enable 2FA on critical accounts like email and banking — it is one of the most effective security measures available.
